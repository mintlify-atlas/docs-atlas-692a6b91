---
title: "Authenticate with the LiteFarm API using JWT tokens"
description: "Obtain a JWT by logging in with email/password or Google SSO, pass it as Authorization: Bearer on every request, and reset passwords via the reset flow."
---

Every request to the LiteFarm API (except login and password-reset endpoints) must include a valid JSON Web Token (JWT) in the `Authorization` header. Tokens are issued at login and signed with HS256. The API server validates the token on every protected request using the `express-jwt` middleware.

## Base URL

```
https://api.litefarm.org
```

---

## Email and password login

Exchange a user's email and password for an access token. The request body must also include the user's `screenSize` object; the server logs this for analytics.

```
POST /login
```

### Request body

<ParamField body="user" type="object" required>
  The user's credentials.
  <Expandable title="properties">
    <ParamField body="email" type="string" required>
      The user's email address. Case-insensitive.
    </ParamField>
    <ParamField body="password" type="string" required>
      The user's plain-text password. The server compares it against the stored bcrypt hash.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="screenSize" type="object" required>
  The user's screen dimensions, used for access logging.
  <Expandable title="properties">
    <ParamField body="screen_width" type="number" required>
      Screen width in pixels.
    </ParamField>
    <ParamField body="screen_height" type="number" required>
      Screen height in pixels.
    </ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="id_token" type="string">
  A signed JWT access token. Pass this as `Authorization: Bearer <id_token>` on subsequent requests.
</ResponseField>

<ResponseField name="user" type="object">
  The authenticated user record from the database.
  <Expandable title="properties">
    <ResponseField name="user_id" type="string">Unique user identifier (UUID).</ResponseField>
    <ResponseField name="email" type="string">User's email address.</ResponseField>
    <ResponseField name="first_name" type="string">User's first name.</ResponseField>
    <ResponseField name="last_name" type="string">User's last name.</ResponseField>
    <ResponseField name="language_preference" type="string">Preferred locale code, e.g. `en`, `es`, `pt`.</ResponseField>
    <ResponseField name="status_id" type="number">Account status. `1` = active, `2` = invited, `3` = expired.</ResponseField>
  </Expandable>
</ResponseField>

### Examples

<CodeGroup>

```bash curl
curl -X POST https://api.litefarm.org/login \
  -H "Content-Type: application/json" \
  -d '{
    "user": { "email": "farmer@example.com", "password": "s3cr3t" },
    "screenSize": { "screen_width": 1440, "screen_height": 900 }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.litefarm.org/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    user: { email: 'farmer@example.com', password: 's3cr3t' },
    screenSize: { screen_width: 1440, screen_height: 900 },
  }),
});
const { id_token, user } = await response.json();
```

</CodeGroup>

### Error responses

| Status | Meaning |
|--------|---------|
| `400`  | Missing `screen_height` or `screen_width` in the request body. |
| `401`  | Password does not match the stored hash. |
| `403`  | No user found for the given email address. |

## Google SSO login

Authenticate using a Google ID token obtained from Google's OAuth 2.0 flow. The server verifies the token with `checkGoogleJwt` and creates the user record automatically if it does not already exist.

```
POST /login/google
```

### Request headers

| Header | Value |
|--------|-------|
| `Authorization` | `Bearer <google_id_token>` — the raw Google-issued ID token. |

### Request body

<ParamField body="language_preference" type="string">
  Preferred locale for the user, e.g. `en`. Used only for new accounts.
</ParamField>

### Response

<ResponseField name="id_token" type="string">
  A LiteFarm JWT access token. Empty string if the Google account maps to an existing password-based account that requires migration.
</ResponseField>

<ResponseField name="user" type="object">
  Basic user info extracted from the Google token.
  <Expandable title="properties">
    <ResponseField name="user_id" type="string">User ID (Google's `sub` claim for SSO users).</ResponseField>
    <ResponseField name="email" type="string">Email from the Google token.</ResponseField>
    <ResponseField name="first_name" type="string">Given name from the Google token.</ResponseField>
    <ResponseField name="language_preference" type="string">Resolved language preference.</ResponseField>
    <ResponseField name="full_name" type="string">Concatenated first and last name.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="isSignUp" type="boolean">
  `true` if the user was created during this request.
</ResponseField>

<ResponseField name="isInvited" type="boolean">
  `true` if the account was in `Invited` status and has pending farm invitations.
</ResponseField>

## Using the JWT token

Once you have an `id_token`, include it in every subsequent API request as a Bearer token:

```
Authorization: Bearer <your_token>
```

### Example authenticated request

<CodeGroup>

```bash curl
curl -X GET https://api.litefarm.org/farm/abc-123 \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```javascript JavaScript
const token = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

const response = await fetch('https://api.litefarm.org/farm/abc-123', {
  headers: { Authorization: `Bearer ${token}` },
});
```

</CodeGroup>

<Note>
  Tokens are signed with HS256. The server uses the `express-jwt` library to validate the signature and expiry on every protected route. A missing or malformed token returns `401 Unauthorized`.
</Note>

### Public endpoints (no token required)

The following paths are exempt from JWT verification:

- `POST /login`
- `POST /user` (account registration)
- `POST /password_reset/send_email`
- `GET /password_reset/validate`
- `PUT /password_reset`
- `PATCH /user_farm/accept_invitation`
- `PATCH /user/accept_invitation`

---

## Token expiry

LiteFarm JWTs have a finite lifetime. When a token expires, the API returns `401 UnauthorizedError`. The client should redirect the user to the login flow to obtain a new token.

<Warning>
  Store tokens securely. Do not persist them in `localStorage` in production browser environments. Prefer `httpOnly` cookies or an in-memory store.
</Warning>

## Password reset flow

The password reset flow uses a dedicated short-lived JWT signed with a separate secret, distinct from the access token secret.

### Step 1 — Request a reset email

```
POST /password_reset/send_email
```

<ParamField body="email" type="string" required>
  The email address of the account to reset.
</ParamField>

The server looks up the account, generates a password-reset token, and sends an email containing a link with `?reset_token=<token>`.

### Step 2 — Validate the reset token

```
GET /password_reset/validate
```

Pass the `reset_token` from the email link as a Bearer token in the `Authorization` header. Returns `200` if the token is valid and unused.

### Step 3 — Set the new password

```
PUT /password_reset
```

Pass the same `reset_token` as the Bearer token. Include the new password in the request body.

<ParamField body="password" type="string" required>
  The new plain-text password. The server hashes it with bcrypt before storing.
</ParamField>

## Check if an email exists

Use this endpoint before directing a user to the login form. It returns account metadata without exposing the password.

```
GET /login/user/:email
```

### Response

<ResponseField name="exists" type="boolean">Whether an active account exists for this email.</ResponseField>
<ResponseField name="sso" type="boolean">Whether the account uses Google SSO.</ResponseField>
<ResponseField name="invited" type="boolean">Whether the account is pending invitation acceptance.</ResponseField>
<ResponseField name="expired" type="boolean">Whether the account requires a password reset.</ResponseField>
<ResponseField name="first_name" type="string">First name of the user, if found.</ResponseField>
